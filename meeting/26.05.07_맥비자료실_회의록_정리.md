## 안건별 정리

### 안건 1. 팀 구조 (정비팀 vs 운영팀 분리)

**논의 내용**

- 김소정: 정비팀에서 카테고리 단일/중복 배치, 시트 분리, 정렬 기준, 태그 활용까지만 합의됐고 실제 진행은 거의 없음.
- 서지연: 자료 정비(기존 자료 정리)와 신규 자료 수급은 성격이 다르므로 두 팀으로 분리 운영 필요.
- 김소정: 초반에는 분리하고, 정비가 끝나면 합류하는 방식 제안.
- 안재찬·서지연·김소정 모두 동의.

**결정 사항**

- **1단계 (현재 ~ 5~6주)**: 전 인원이 함께 **기존 자료 정비**에 집중
- **2단계 (정비 완료 후)**: 정비팀·운영팀 구분 없이 모두 **운영(신규 자료 수급·관리)으로 전환**
- 정비 단계 동안에는 팀 구분이 사실상 의미 없음 — 모두 검수 분담에 참여

### 안건 2. 기존 자료 정비 방식

**논의 내용**

- 안재찬: AI(Claude)로 1차 자동 정비 완료. 비어있는 행, 404 응답, 서비스 종료된 자료를 자동 제거. 약 200건은 사람 검수가 필요해 "보류" 처리.
- 안재찬: 822건이 자료실로 자동 등록되도록 분류 완료. 카테고리 합치기, 자료 형식 자동 분류(가이드/템플릿/아티클 등), 발행일 메타데이터 추출도 1차 완료.
- 서지연: AI는 링크 유효성만 신뢰 가능. 발행일·시의성·"지금도 유효한지" 같은 항목은 사람의 매뉴얼 검수가 필요.
- 임종헌: 기존 시트 형태가 들쑥날쑥(어떤 행은 텍스트만, 어떤 행은 링크 10개 등). 카페·로그인 필요 콘텐츠도 섞여 있음.
- 안재찬: 안 열리는 링크는 일괄 제외. 정비팀이 새로운 자료를 발굴하는 것까지는 범위 외.

**결정 사항**

- AI 1차 정비 결과를 사람이 매뉴얼 검수
- 안 열리는 링크, 텍스트만 있고 의미 없는 행은 삭제
- 의미 있는 텍스트 자료는 FAQ로 별도 분류 (안건 3)
- **검수 대상**: 자료 DB(822건) + 보류 큐(74건) + broken 시트 모두 포함

**액션 아이템**

- ~~안재찬: 시트 번호 누락 부분 정리(81·88 등)~~ — **확인 완료** (누락된 번호는 broken 시트로 분리되어 있음)
- 서지연: 정비 작업용 시트 사본 생성 + 운영진 편집 권한 부여 — **완료**

### 안건 3. 자료 분류 — 자료실 / 인사이트 / FAQ 3개 메뉴

**논의 내용**

- 안재찬 제안: 자료를 3개 카테고리로 구분
    - **자료실**: 파일 링크(PDF·PPT·구글 드라이브 자료)
    - **인사이트**: 외부 링크(아티클·영상·노션)
    - **FAQ**: Q&A 형식의 텍스트 자료 (별도 메뉴)
- 임종헌: 인사이트에 텍스트가 많이 들어가면 페이지 로딩이 느려질 우려. Supabase 같은 DB로 분리하는 방안 검토.
- 안재찬: Apps Script + 스프레드시트 연동 구조라 로딩 지연이 발생. 추가 비용 발생을 허용해야 근본 해결 가능.
- 임종헌: Supabase 무료 tier(500MB)면 텍스트 데이터로는 충분.
- 서지연: Supabase 무료는 일주일 비활성 시 프로젝트 일시 중지(pause) 발생. 사용자 접속이 일주일 안에 한 번이라도 있으면 유지됨.
- 서지연: FAQ까지 살리면 자료실 활성화에 도움 + 맥비님이 강조한 "정말 인사이트 있는 공간"에 부합.

**결정 사항**

- FAQ를 별도 메뉴(탭)로 신설
- 기존 Q&A 텍스트 자료를 정제해서 FAQ로 이관
- 인사이트는 외부 링크 위주, FAQ는 텍스트 위주로 깔끔하게 분리

**액션 아이템**

- 임종헌: FAQ 샘플 정리 → 카톡방 공유 → 의견 수렴 후 본격 진행 — 다음 회의 전

### 안건 4. 신규 자료 등록 프로세스

**논의 내용**

- 안재찬 시연: AI(Gemini) 연동 등록 흐름. URL 1개 붙여넣으면 제목·요약·분류·자료 형식 자동 추출 → 운영자 2명 승인 시 자료실 자동 이관.
- 서지연: 등록 폼이 잘 만들어졌으니 운영팀뿐 아니라 멤버에게도 오픈 가능.
- 안재찬: DB만 직접 건드리지 않게 하면 멤버 오픈 가능.

**결정 사항**

- 등록 페이지를 멤버에게 오픈 (URL만 알면 누구나 등록 가능)
- 운영자 2명 승인 후 자료실 노출 — 시스템상 이미 구현됨

### 안건 6. 정비 일정 및 인력 분담

**논의 내용**

- 정비 대상: 자료 DB + 보류 큐 + broken 시트 = 약 900건+
- 1인당 주 20개 페이스
- 8명 × 주 20개 = 주 160개 → **5~6주 소요**
- 번호 단위로 분담(예: 1~80번 / 81~160번 / ...)
- 시트는 **서지연이 별도 사본 생성**(완료), 정비 완료 후 원본에 덮어쓰기

**결정 사항**

- 전 인원 **8명**이 함께 정비 진행 (김다슬 퇴사로 9명 → 8명)
- **1인당 주 20개**, **번호 구간으로 분담**
- 5~6주 후 다음 회의에서 중간 점검

**액션 아이템**

- 서지연: 인원별 번호 구간 매핑 표 정리 후 공유 — 즉시
- 운영진 전원: 1인 주 20개 검수 — 6주간 지속
