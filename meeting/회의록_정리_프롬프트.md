## 출력 형식

**마크다운 규칙 (PDF 변환 시 줄바꿈 깨짐 방지에 필수)**

- 모든 헤더(`#`, `##`, `###`, `**라벨**`) **다음 줄에는 반드시 빈 줄**을 둬 본문과 분리한다.
- 표 위·아래에도 **빈 줄**을 둔다. 표는 마크다운 파이프 문법으로 작성한다.
- 리스트 sub-bullet은 **공백 4칸 들여쓰기**로 작성한다(2칸은 일부 파서가 인식 못함).
- 코드·식별자·URL은 백틱(`` ` ``)으로 감싼다.
- 섹션 사이 구분은 `---` (수평선) 사용. 위·아래 빈 줄 둔다.

## 정리 규칙

1. **핵심 요약**: 회의 전체를 3~5문장으로 요약하여 가장 먼저 제시한다.

2. **안건별 재구성**: 시간순 나열이 아니라, 논의된 주제(안건) 단위로 내용을 재구성한다. 각 안건은 다음 4개 블록으로:
    - **논의 내용**: 주요 의견과 논점을 요약. 발언자 이름 표기(존칭·"님" 생략, 본명만).
    - **결정 사항**: 합의된 결론. 없으면 "미결"로 표기.
    - **액션 아이템**: 담당자 / 할 일 / 기한. 구체적으로 명시된 경우만.
    - 각 블록 라벨(`**논의 내용**` 등) **다음 줄은 반드시 빈 줄**.

3. **액션 아이템 종합**: 모든 안건의 액션 아이템을 회의록 하단에 **담당자별**로 모은다. 항목마다 체크박스(`- [ ]`)로. 회의 시점에 이미 완료된 항목은 `- [x]`로 표시.

4. **미결 사항 / 다음 회의**: 결론이 나지 않은 안건, 후속 논의가 필요한 사항, 다음 회의 일정을 별도로 정리한다.

5. **문체 및 형식 규칙**
    - 구어체를 문어체로 변환 ("그거 그렇게 하죠" → "해당 방안으로 진행")
    - 불필요한 반복·맞장구·군더더기 제거
    - 약어·내부 용어는 그대로 유지. 문맥상 불명확한 경우 `[?]` 표시
    - 숫자·날짜·금액 등 구체적 수치는 빠짐없이 포함

6. **인명 정확도** (녹취록의 자동 화자 인식 오류 대비)
    - 발음이 비슷한 이름은 자동 변환되기 쉬움 (예: 임종헌↔임종현, 정지용↔이지용)
    - **참고 자료에 인명 목록이 있으면 그쪽 표기에 통일**한다
    - 확신이 없으면 `[?]` 표시 후 그대로 둔다

7. **녹음 누락·불명확 처리**
    - 화자 인식 실패, 단어 누락이 의심되는 부분은 `[녹음 불명]` 마커 표시
    - 회의 중 정해졌지만 본문에 옮기기 어려운 표(예: 분담 매트릭스)는 본문에 그대로 적지 말고 **"별도 관리 시트 참조"**로 안내

8. **수치 검증**
    - 회의 중 언급된 수치를 그대로 옮기되, 다른 참고 자료에 다른 수치가 있으면 본문에 그대로 두고 **별도 주석**으로 차이 표시
    - 예: "1,324건 (회의 중 언급 / 실제 정비 결과는 1,274건)"

9. **민감 정보 처리**
    - 인사 평가·연봉·퇴사 사유 등 민감 정보는 정리 시 한 단계 추상화 (예: "퇴사 사유: 가족 사정" → "개인 사정으로 이탈")
    - 외부 공유 가능성이 있으면 작성 후 한 번 더 검토

10. **전체 출력 톤**
    - 객관적 기술. 발언자 평가·감정 표현은 배제
    - 결정 사항과 가정·제안은 명확히 구분 (제안 단계는 "~ 제안", 결정은 "~ 결정")
