# 운영팀 효율화 제안 v1 — 비용 0 기반

작성: 2026-05-05 / 운영팀 안재찬
대상: 운영팀 내부 검토용 초안

---

## 0. 한 줄 요약

**Google Workspace(시트 + 드라이브 + 폼 + Apps Script)만으로** 링크와 실파일을 함께 아카이빙하는 단일 진실의 원천(SSOT) 체계를 만들고, 멤버 제안 → 운영팀 검토 → 일괄 등록 루틴을 정착시킵니다.

## 1. 왜 노션·Supabase가 아닌 "구글 스택"인가

회의록(2025-12-21, 2026-01-29)에서 노션 DB → 추후 Supabase 이전이라는 phase 안이 우세했지만, 운영팀 1차 합의는 "**비용 0**"입니다. 이 제약을 우선시하면 다음과 같이 정리됩니다.

| 도구 | 비용 | 파일 아카이빙 | DB/검색 | 자동화 | 평가 |
|---|---|---|---|---|---|
| **Google Sheets + Drive + Forms + Apps Script** | 무료(15GB) | O (Drive) | O (필터·슬라이서) | O (Apps Script) | ✅ 1차 합의에 부합 |
| Notion 무료 | 무료 | △ (5MB/파일 제한) | O | △ (외부 도구 필요) | 파일 아카이빙 부적합 |
| Supabase + 자체 웹 | 무료 tier | O | O | O | 개발 리소스 필요 |

**핵심**: "파일까지 다운로드 가능한 아카이빙"이 요구사항이므로 노션 무료 플랜은 부적합합니다. 구독팀이 유료 플랫폼을 만들 때 이전을 검토하면 충분합니다.

## 2. 시스템 아키텍처

```
[멤버]                      [운영팀]                    [이용자]
   │                            │                          │
   │ 구글폼 제안                │                          │
   │ (URL or 파일)              │                          │
   ▼                            ▼                          │
┌─────────────┐          ┌──────────────┐                 │
│ 제안 응답   │  주1회   │  SSOT 시트   │  ◀──── 조회 ───┤
│ 시트(폼)    │ ───────▶ │ (자료 DB)    │                 │
└─────────────┘  검토    └──────────────┘                 │
                                │                          │
                                │ 파일 자동 분류           │
                                ▼                          │
                         ┌──────────────┐                 │
                         │ Drive 폴더   │  ◀──── 다운 ───┘
                         │ (카테고리별) │
                         └──────────────┘

       [Apps Script]
       - 주1회 링크 유효성 점검 → "점검필요" 상태 표시
       - 폼 응답 → SSOT 자동 임시 등록(승인 전 staging)
       - 주간 백업 스냅샷 → 백업 폴더
```

## 3. SSOT 시트 정비 (현재 V3.1 기준)

현재 v2 파일에는 V3.1 / V2.2 / V2.1 / 카테고리 시트가 공존합니다. **V3.1을 SSOT로 격상**하고 나머지는 `_archive_` 시트로 이동시킵니다.

### 컬럼 정의 (제안)

| # | 컬럼 | 타입 | 필수 | 설명 |
|---|---|---|---|---|
| 1 | No | 자동 | ✅ | 시트 함수로 자동 채번 |
| 2 | 대분류 | dropdown | ✅ | 카테고리 마스터 시트 참조 |
| 3 | 소분류 | dropdown | ✅ | 대분류에 종속 |
| 4 | 태그 | multi | - | `#기획서 #실무 #B2B` 형태, 검색용 |
| 5 | 자료 제목 | text | ✅ | |
| 6 | 보조설명 | text | - | 한 줄 요약 |
| 7 | 외부 링크 | url | △ | 외부 자료일 때 |
| 8 | 파일 링크 | url | △ | Drive 파일 직링크 (실파일 아카이빙) |
| 9 | 자료 형식 | dropdown | ✅ | 아티클 / 영상 / 기획서 / 가이드 / 템플릿 / 세미나 |
| 10 | 발행일 | date | - | 원본 발행 시점 |
| 11 | 등록일 | date | ✅ | 자료실 등록 시점 |
| 12 | 등록자 | text | ✅ | |
| 13 | 상태 | dropdown | ✅ | 공개 / 점검필요 / 비공개 / 삭제예정 |
| 14 | 마지막 점검일 | date | - | Apps Script가 갱신 |
| 15 | 카테고리 담당 | text | - | 카테고리 마스터에서 자동 매핑 |
| 16 | 노출 등급 | dropdown | ✅ | 무료 / 유료(구독팀 협업) |
| 17 | 비고 | text | - | |

### 시트 구성

- `자료 DB` — SSOT (위 컬럼)
- `카테고리` — 대/소분류 마스터 + 담당자 매핑 (현 시트 정비)
- `_staging_` — 폼으로 들어온 미승인 제안 (운영팀 검토 대기열)
- `_archive_v2.x` — 구버전 백업 (참조용, 편집 금지)
- `_점검로그` — Apps Script 자동 기록 (깨진 링크, 점검 이력)

> **단일 시트 + 필터/슬라이서 vs 카테고리별 탭 분리**: 회의록에서는 탭 분리 의견이 많았으나, 운영 측면에서 단일 시트 + 슬라이서/필터뷰가 압도적으로 효율적입니다(중복 배치 시 동기화 이슈 없음, 정렬·검색 일관성). 이용자에게는 슬라이서로 카테고리 뷰를 즉시 전환하도록 제공.

## 4. Drive 폴더 구조 — 실파일 아카이빙

**공유 드라이브** 또는 운영팀 공용 폴더(공유 권한 부여) 하나에 다음 구조로 구성:

```
[맥비기획_자료실]
├── 00_운영
│   ├── 자료실_DB           ← Google Sheets (SSOT)
│   ├── 운영가이드.gdoc
│   └── _백업/              ← 주간 자동 스냅샷 (Apps Script)
├── 01_면접·채용·이직
├── 02_기획·PM
├── 03_UX·디자인
├── 04_개발·기술
└── 05_커리어
```

운영 규칙:
- 파일명 규칙: `[YYYY-MM-DD]_자료제목_등록자.확장자`
- 폼 업로드 → Apps Script가 대분류에 따라 해당 폴더로 자동 이동 + 시트의 "파일 링크" 컬럼에 직링크 채움
- 저작권 이슈가 있는 외부 자료는 **링크만 보관**, 파일은 업로드하지 않음 (자료 형식 = 아티클/영상은 파일 업로드 비활성)

## 5. 신규 자료 등록 워크플로우

### A. 멤버 제안 경로

1. 멤버가 구글폼 작성 (10초 안에 제출 가능하게)
   - URL **또는** 파일 업로드 (택1)
   - 추천 카테고리 (선택)
   - 한 줄 설명
   - 닉네임 (리워드용)
2. 응답이 `_staging_` 시트로 누적
3. **주 1회 운영팀 정기 검토**(요일 고정 권장: 일요일 저녁)에서 승인/반려
4. 승인 항목만 SSOT로 이관 + 메타데이터 보강

### B. 정비팀 인계 경로

1. 정비팀이 분류 완료한 자료 묶음을 **인계 시트**(공유)로 전달
2. 운영팀이 컬럼 매핑 후 SSOT로 일괄 import
3. 파일이 있으면 카테고리 폴더에 업로드

### C. 운영팀 직접 등록

- 카테고리 담당자가 본인 카테고리 자료 발견 시 즉시 등록
- 등록 시 "등록자" 컬럼에 본인 표시 (기여도 측정용)

## 6. 카테고리 담당제 (회의록 합의 사항)

현재 카테고리 시트에 일부만 매핑된 담당자를 **전 카테고리에 채웁니다**.

담당자 책임:
- **월 1회 본인 카테고리 자료 점검**(시의성·중복·링크 오류)
- 카톡방에서 본인 카테고리 관련 자료 발견 시 등록 또는 \_staging\_에 기록
- 멤버 신고 시 1차 대응

자동화 보조:
- Apps Script가 "점검필요" 상태인 자료를 매주 월요일 운영진 카톡방으로 요약 (Webhook)
- 깨진 링크는 자동으로 상태 변경 + 담당자 멘션 메시지 생성

## 7. 단계별 실행 계획

| Phase | 기간 | 목표 | 산출물 |
|---|---|---|---|
| **0. 정책 합의** | 1주 | 본 제안서 운영팀 합의, 정비팀·구독팀 얼라인 | 합의된 v2 제안서 |
| **1. 환경 세팅** | 1주 | Drive 폴더, SSOT 시트 v3 스키마, 구글폼, 기본 Apps Script | 시스템 가동 가능 상태 |
| **2. 파일럿** | 2주 | 50~100건 실제 등록·검토·점검 사이클 1회전 | 소요시간/병목 보고서 |
| **3. 전체 이관** | 4주 | 정비팀 정비분 일괄 import, 카테고리 담당자 분담 | SSOT 정상 운영 |
| **4. 정착·고도화** | 지속 | 월간 점검, 링크 자동 점검, 리워드 운영 | 월간 운영 리포트 |

## 8. 비용 0 이후 확장 옵션 (참고용, 즉시 결정 불요)

- **Notion 공개 페이지**: 시트 → 노션 임베드/연동으로 이용자용 view 페이지만 제공 가능
- **Supabase + 간단 웹**: 구독서비스 런칭 시점에 검색 UI 강화 (회의록의 phase 2)
- **AI 자동 분류**: Apps Script + Gemini 무료 API로 URL 메타데이터·카테고리 자동 추천

## 9. 회의록과의 정합성 체크

| 합의/논의 사항 | 본 제안 반영 여부 |
|---|---|
| 등록 권한 오픈 폐지 → 운영팀 등록 | ✅ |
| 멤버는 폼으로 제안 | ✅ |
| 주 1회 일괄 등록 | ✅ |
| 카테고리별 담당자제 | ✅ (마스터 시트 빈칸 채우기) |
| 중복 배치 vs 단일 배치 | 단일 배치 + 태그 다중으로 검색 대응 |
| 카테고리별 탭 분리 | 슬라이서/필터뷰로 대체 (관리 효율 우선) |
| 링크 유효성 자동 점검 | ✅ (Apps Script) |
| 백업 | ✅ (주간 스냅샷) |
| 자료 형식 표시 | ✅ (자료 형식 컬럼) |
| 무료/유료 구분 | ✅ (노출 등급 컬럼, 구독팀 협업) |
| 파일 아카이빙 | ✅ (Drive + 파일 링크 컬럼) |

## 10. 결정 필요 사항 (운영팀 합의 항목)

- [ ] 기본 정렬 기준: 등록일 최신순 (제안)
- [ ] 주 1회 정기 검토 요일·시간
- [ ] 카테고리 담당자 빈칸 채우기 (현재 5개 비어있음)
- [ ] 공유 드라이브 소유 계정 (개인계정 vs 운영팀 공용계정 신설)
- [ ] 멤버 리워드 정책 — "월간 베스트 제안자" 등 (구독팀과 협의)
- [ ] 정비팀 → 운영팀 인계 시트 양식

## 부록 A. 운영 리소스 추정

| 작업 | 빈도 | 1회 소요 | 담당 |
|---|---|---|---|
| 폼 응답 검토·승인 | 주 1회 | 30~60분 | 당번제 (운영팀 N명 로테이션) |
| 카테고리 점검 | 월 1회 | 30분/카테고리 | 카테고리 담당자 |
