## 검토 기준

1. **비용 0** (1차 합의 사항)
2. **등록 간소화** — URL/파일만 던지면 끝
3. **AI 자동 분류** — 카테고리·태그·한줄설명 자동 생성
4. **운영팀 부담 최소화** — 검토만 하면 되는 구조
5. **모바일 친화** — 카톡방 멤버가 모바일에서 즉시 등록

## 채택: 옵션 A — Google Apps Script Web App + Gemini

**한 줄로**: "Apps Script로 만든 모바일 친화 등록 페이지 + Gemini가 자동으로 메타데이터를 채워주고 → staging 시트로 저장 → 운영팀은 승인 버튼만"

### 왜?
- **0원**: Apps Script 무료 + Gemini Flash 무료 tier(분당 백만 토큰, 일 1500회)로 충분
- **단일 URL 1개로 등록 가능**: 모바일에서 카톡 → 링크 복붙 → 제출 끝
- **AI가 다 채움**: 제목·요약·카테고리·태그·자료형식까지 자동
- **운영 부담 최소**: 검토 화면에서 ✓/✗ 버튼만, 필요시 한 줄 수정
- **카톡방 친화**: 단축 URL 하나만 공지하면 됨, 별도 앱 설치 불필요

### 추가 채널 (선택)
- 운영팀 한정으로 `n8n` Webhook 연결 가능 (정지용 제안 그대로 살림)
- 카톡방에 봇 셋업하려면 별도 phase에서 진행

## 사용자 등록 흐름 (UX)

```
[모바일/PC 브라우저]
1. 단축 URL 접속
2. URL 입력칸 1개 + 닉네임
3. "분석" 버튼
   ↓ (1~3초)
4. AI가 자동으로 채운 결과 표시:
   - 제목, 한 줄 요약, 대분류, 소분류, 태그, 자료 형식
5. 사용자는 그대로 두거나 한 줄 수정 가능
6. "제출" 버튼 → staging 시트 저장
7. "제안 완료, 운영팀 검토 후 자료실에 반영됩니다"
```

총 입력 = URL + 닉네임 (필요시 한 줄 수정).

## 운영팀 검토 흐름 (UX)

```
[관리자 페이지]
1. staging 시트의 미검토 항목 카드 형태로 노출
2. AI 분석 결과 + 원문 미리보기
3. 카드별 [승인] [반려] [수정 후 승인] 버튼
   - 승인: SSOT 시트로 이관 + Drive에 파일 있으면 카테고리 폴더로 이동
   - 반려: staging에 사유 기록
4. 일괄 승인도 가능 (체크박스)
```

목표 처리 시간: **건당 10초 이내, 주 1회 30~60분 이내 완료**.

## 자동화 (Apps Script Triggers)

| 작업 | 빈도 | 설명 |
|---|---|---|
| 링크 유효성 점검 | 매주 월요일 새벽 | SSOT 전체 URL HEAD 요청, 깨진 링크 → 상태 "점검필요" + 카테고리 담당자 멘션 메시지 생성 |
| 백업 스냅샷 | 매주 일요일 새벽 | SSOT 시트 복사본을 `_백업` 폴더에 타임스탬프와 함께 저장 |
| Staging 알림 | 매일 저녁 | 오늘 들어온 staging 건수 카톡 메시지 텍스트 생성 (운영진이 복붙) |

## 비용 시뮬레이션 (Gemini Flash 기준)

- 1건 분류당 평균 **약 1,500 토큰** (입력 본문 발췌 + 출력 JSON)
- Gemini 2.0 Flash 무료 tier: 1,500 RPD (요청/일), 1M TPM (토큰/분)
- 하루 1,500건까지 무료 → 자료실 수요 (월 100~300건) 대비 **충분히 여유**
- 만약 한도 초과시 자동으로 휴리스틱 fallback (페이지 메타태그 기반 추론)

## 필요한 권한·계정 (사용자 액션)

1. **Gemini API 키** (https://aistudio.google.com → Get API key, 무료, 5분)
2. **Google 계정** — Apps Script 배포에 필요 (운영팀 공용 계정 권장)
3. **공유 Drive 폴더** — 파일 아카이빙용, 공용 계정으로 생성 후 폴더 ID 전달
4. **자료실 시트 편집권** — 이미 보유

이 4가지가 준비되면 Apps Script 프로젝트에 키/ID만 채워 넣고 배포(Deploy → Web app)하면 즉시 동작.
