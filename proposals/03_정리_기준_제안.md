## 1. 자료 필터링 기준

### 1-A. 즉시 제거 (자동 판단)

| 조건 | 사유 |
|---|---|
| 링크 + 제목 모두 비어있음 | 데이터 가치 0 |
| "(원글의 링크삭제 됨)" 마커 포함 | 원본 소실 |
| "서비스 종료됨" 마커 포함 | 자료 가치 상실 |
| 단축 URL(`bit.ly` 등) 중 HTTP 응답 4xx/5xx | 만료된 단축 링크 |
| 노이즈 카테고리(`17-1`, `23-1`, `34-1`, `43-1`, `링크오류 3` 등) | 색인 마커일 뿐 |

### 1-B. 보류 (사람 검토 필요)

| 조건 | 처리 |
|---|---|
| 링크는 살아있으나 발행일 없음 + 제목에 시의성 키워드(연도, "최신", "트렌드 20XX") | "시의성 만료 후보" 태그를 달고 운영팀이 1차 검토 |
| 제목이 매우 짧거나 의미 모호 ("기획", "참고" 등 5자 이하) | 본문 메타에서 보강 가능하면 유지, 안 되면 제거 |
| Q&A 형식의 톡방 임시 노트 (`기획이슈_*`, 질문이 카테고리로 들어간 87건) | **별도 "기획 Q&A" 섹션**으로 분리 또는 일괄 제거 |

### 1-C. 유지 기준 (자동 import)

- 살아있는 링크 + 제목 있음
- 영구가치 자료(방법론·기본기·법규·도구 사용법·실무가이드)
- 시의성이 있더라도 발행일이 명확한 자료 (트렌드 분석 등)

### 결과 추정

- 즉시 제거: ~150건 (Q&A 형식 87건 + 노이즈 카테고리 약 70건)
- 시의성 검토 후보(보류): ~200건
- 자동 유지(import): **~900~950건**

## 3. 파일 네이밍 규칙

운영팀이 직접 업로드하는 파일(macbee-document·향후 등록 분)에 일관된 규칙 적용.

### 3-A. 양식

```
[카테고리코드]_[자료명_정제]_v[버전]_[YYMM].확장자
```

- **카테고리코드**: `01-면접` / `02-기획PM` / `03-UX디자인` / `04-개발기술` / `05-커리어` / `06-비즈마케팅` / `99-기타`
- **자료명_정제**: 공백 → 언더스코어, 괄호·점·특수문자 제거, 한글·영문·숫자·하이픈만 허용
- **버전**: 알면 명시, 모르면 `v1`
- **YYMM**: 알려진 발행일(yyMM). 모르면 생략 가능
- 확장자: 소문자
- 전체 80자 이내

### 3-C. 자동 변환 가능

`importMacbeDocs()`에서 파일을 카테고리 폴더로 옮길 때 위 규칙으로 **자동 rename** 처리하도록 코드에 반영. 사용자가 손으로 바꿀 필요 없음.

## 4. 발행일 처리

| 케이스 | 처리 |
|---|---|
| 외부 URL meta(og:article:published_time)에 있음 | 자동 추출 |
| URL에 연도 패턴(`/2024/...`) 발견 | 추정 발행연월 (시트의 `발행일(추정)` 컬럼에 별도 표시) |
| 파일 메타데이터 `Created`/`Modified` | macbee-document 파일은 Drive의 `created` 사용 |
| 모두 없음 | 빈값 → 카드에 "발행일 미상" 표기 (정렬은 등록일 기준) |
